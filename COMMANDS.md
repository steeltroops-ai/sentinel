# Command Reference
**Task Runner | python run.py <command>**

---

## The Command Problem

Makefiles break on Windows. Shell scripts are not portable. Docker Compose does not handle Python environments. The solution is a single Python task runner that works everywhere and handles dependencies correctly.

Every command below runs through `run.py`. No make. No bash. No platform-specific nonsense.

---

## Essential Commands

**Setup:**
```bash
python run.py install      # Install dependencies via uv
```

**Data generation:**
```bash
python run.py data         # Generate 5000 profiles (fraud_ratio=0.4)
```

**Testing:**
```bash
python run.py test         # Run all 82 tests
```

**Training (all with MLflow tracking):**
```bash
python run.py train-fast   # 1k episodes, no tracking (5 min)
python run.py train        # 3k episodes, tracked (15 min)
python run.py train-full   # 10k episodes, tracked (45 min)
```

**Docker services:**
```bash
python run.py docker-up    # Start all 9 signal services + orchestrator
python run.py docker-down  # Stop all services
```

**Documentation:**
```bash
python run.py update-docs      # Copy artifacts to docs, update README
python run.py visualize-mlflow # Generate training comparison plots
python run.py mlflow-ui        # Launch MLflow UI at localhost:5000
```

**Submission:**
```bash
python run.py submit-check # Verify all deliverables exist
```

---

## Training Commands Explained

**train-fast:** 1000 episodes, no MLflow. Use for rapid iteration during development. Completes in 5 minutes. Does not log to mlruns/.

**train:** 3000 episodes, MLflow enabled. Standard training run. Logs metrics, hyperparameters, and convergence criteria to mlruns/. Completes in 15 minutes.

**train-full:** 10000 episodes, MLflow enabled. Production training run. Maximizes convergence quality. Logs everything. Completes in 45 minutes.

All training commands save artifacts to `artifacts/training/`:
- `convergence_report.json` - Final metrics (FN/FP rates, probe usage, convergence status)
- `learning_curve.png` - Reward progression over episodes
- `episode_traces.png` - Sample decision sequences
- `learning_curve.csv` - Raw episode data for analysis

## Quick Workflows

**First time setup:**
```bash
python run.py install
python run.py data
python run.py test
```

**Development iteration:**
```bash
python run.py format
python run.py test
python run.py train-fast
```

**Final submission:**
```bash
python run.py clean
python run.py data
python run.py train-full
python run.py update-docs
python run.py submit-check
```

**MLflow tracking:**
```bash
python run.py train-full          # Train with tracking
python run.py visualize-mlflow    # Generate comparison plots
python run.py mlflow-ui           # View runs in browser
```

## Artifact Locations

**Training outputs** (gitignored):
- `artifacts/training/` - Learning curves, traces, reports, models
- `mlruns/` - MLflow experiment tracking data

**Documentation** (committed):
- `docs/learning_curve.png` - Copied from artifacts
- `docs/episode_traces.png` - Copied from artifacts
- `docs/training_summary.png` - Generated by visualize-mlflow
- `docs/signal_distributions.png` - Generated by notebooks

**Data** (gitignored):
- `data/synthetic_profiles.json` - Generated profiles
- `data/signal_distributions.csv` - Signal statistics

Run `python run.py update-docs` after training to copy artifacts to docs/ and update README with latest metrics.
